# Phase 1 セキュリティ対策必須項目の追加

**日付**: 2025-03-20  
**対象**: `docs/memory/backend/backend.md`

## 変更内容

backend.md のタスク一覧に **「7. セキュリティ対策（Phase 1 必須）」** を追記した。

## 追加した項目（5つ）

| 項目 | 概要 |
|------|------|
| 入力バリデーション（Zod 等） | frames・userHint・prompt のスキーマ検証、型・長さ・枚数制限 |
| ペイロードサイズ制限 | base64 画像の合計サイズ制限、過大リクエストの早期リジェクト |
| エラーハンドラ（詳細を非表示） | 本番でスタックトレースを返さない、汎用メッセージのみ |
| CORS 設定 | 許可オリジンを明示、ワイルドカード不使用 |
| GEMINI_API_KEY のシークレット管理 | wrangler secret で設定、Env 型で型定義 |

## 理由

- Phase 1 の API 実装と並行して、最小限のセキュリティ対策を組み込むため
- 入力検証・ペイロード制限は不正リクエストや API 濫用の防止に必須
- エラー詳細の非表示は情報漏洩リスクの低減
- CORS・シークレット管理はデプロイ前の必須設定
