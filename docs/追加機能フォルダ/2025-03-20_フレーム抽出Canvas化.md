# フレーム抽出: ffmpeg.wasm から Canvas API へ変更

日付: 2025-03-20

## 目的

ブラウザ内の ffmpeg.wasm は動画全体のデコード負荷が高く、長時間動画で極端に遅くなる。  
**HTMLVideoElement + Canvas** でキーフレーム時刻にシークして JPEG を切り出す方式に切り替え、高速化と依存削減を図る。

## 変更内容

| 項目 | 内容 |
|------|------|
| `frontend/src/lib/frameExtractor.ts` | ffmpeg 廃止。`seek` → `drawImage` → `toDataURL('image/jpeg')` で base64 配列を生成 |
| `frontend/src/components/VideoUploader.tsx` | `loading_ffmpeg` 状態を削除（メタデータ → 抽出の2段階） |
| `frontend/vite.config.ts` | `@ffmpeg` 用の `optimizeDeps.exclude` と COOP/COEP ヘッダーを削除（WASM 用だったため） |
| `frontend/package.json` | `@ffmpeg/ffmpeg` / `@ffmpeg/util` を削除 |

## 挙動

- フレーム枚数: 既存どおり 3〜5（既定 4）
- タイムスタンプ: `duration / (count + 1) * (i + 1)` で均等配置
- 解像度: 長辺最大 1280px にスケール（負荷とペイロード削減）
- 進捗: 抽出枚数ベースで `onProgress`（25% / 50% / 75% / 100% など）

## 注意

- 対応コーデックは **ブラウザの動画デコーダに依存**（H.264 の mp4 は一般的に可）
- CORS は blob URL 利用のため通常問題なし

## 検証

- `cd frontend && npm run build`
- ローカルで動画アップロード → フレームプレビュー → 分析まで E2E
