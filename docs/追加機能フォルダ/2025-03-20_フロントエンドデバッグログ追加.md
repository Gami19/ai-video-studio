# フロントエンド デバッグログ追加

日付: 2025-03-20

> **追記**: フレーム抽出は後日 **Canvas API** に切り替えた。`frameExtractor` のログは FFmpeg ではなく Canvas 抽出（メタデータ・シーク・描画）向け。詳細は `2025-03-20_フレーム抽出Canvas化.md`。

## 概要

フレーム抽出・API・画面フローがどこで止まっているか切り分けしやすいよう、コンソールログを追加した。

## 追加・変更ファイル

| ファイル | 内容 |
|----------|------|
| `frontend/src/lib/debugLog.ts` | `debugLog` / `debugElapsedMs` / `isDebugEnabled` |
| `frontend/src/lib/frameExtractor.ts` | Canvas 抽出時のメタデータ・シーク・描画・完了ログ（※後日 Canvas 化） |
| `frontend/src/components/VideoUploader.tsx` | ファイル選択、状態、エラー |
| `frontend/src/lib/api.ts` | `/api/analyze`・`/api/generate` の開始・成功・失敗と所要時間 |
| `frontend/src/App.tsx` | analyzing / generating / done / error の遷移 |
| `frontend/.env.example` | 任意の `VITE_DEBUG=true` 説明 |

## 有効条件

- **`npm run dev`**: 常にログ出力
- **本番ビルド**: 既定では出力なし。必要なら `VITE_DEBUG=true` を環境変数に設定

## コンソールの見方

- フィルタに **`ai-video-studio`** と入力すると当該ログだけ表示しやすい
- **`[timing]`** で各フェーズの経過時間（ms）を確認できる
- **メタデータ待ちのログが出ない** → 動画読み込み前で停止
- **`Canvas 抽出: シーク＆描画開始` の後に進まない** → シークまたはデコードで停止の可能性
- **`Canvas 抽出ループ` の timing が長い** → 長尺動画・高解像度・シーク回数に比例
